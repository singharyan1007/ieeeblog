# Git and GitHub: A Comprehensive Guide

Git and GitHub are essential tools for version control and collaborative software development. In this blog post, we will explore the fundamental concepts of Git and GitHub, including the anatomy of a Git repository, file management, and common operations like adding, staging, committing, and pushing files.

## Anatomy of a Git Repository

When you initialize a directory as a Git repository, it creates a hidden folder called `.git`. This folder contains critical information that Git uses to manage version control. Let's delve into the contents and purpose of the `.git` folder:

1. **Objects**: This directory stores the actual file content and metadata, including commits, trees, and blobs.
2. **Refs**: Here, Git stores references to commit objects, such as branch heads, tags, and remote references.
3. **HEAD**: It points to the currently checked out branch, indicating which branch or commit you are currently working on.
4. **Index**: Also known as the staging area, it keeps track of the changes you want to include in the next commit.
5. **Config**: This file stores configuration settings for the Git repository, such as user information and remote repository URLs.
6. **Hooks**: Git allows you to run custom scripts before or after certain actions, and this directory contains the scripts for these hooks.
7. **Logs**: Git maintains logs of various actions and events in this directory, providing a historical record of changes.

## Adding a File

To add a file to your Git repository, you can use the following command: `git add <file_name>`.

This command stages the specified file, preparing it to be included in the next commit.
It is also referred as `Staging a File`

The staging area allows you to select specific changes or files to be included in a commit. To stage a file, use the following command:

```bash
git add <file_name>
```
This command adds the changes in the specified file to the staging area.
## Committing a File

Once you have staged the desired changes, you can commit them to the repository. Use the following command to create a commit:

```bash
git commit -m "Commit message"
```
Replace "Commit message" with a descriptive message explaining the changes made in the commit.

## Unstaging a File

If you have staged a file but wish to remove it from the staging area without losing the changes, you can use the following command:

```bash
git reset <file_name>
```
This command removes the specified file from the staging area while preserving the changes.
## Deleting a Commit

In certain cases, you might need to delete a commit from the Git history. Use the following command to remove the most recent commit:

```bash
git reset HEAD~1
```

## Pushing a File

To push your committed changes to a remote repository, use the following command:

```bash
git push origin <branch_name>
```
Replace `<branch_name>` with the name of the branch where you want to push the changes.

## Stashing a file
When you want to work on some feature on the base project without commiting the previous changes or features then the better approach to achieve this is to send the staged file to the stash area and it can be done using:
```bash
git stash
```
and to get the file back from the stash area we use the following command:
```bash
git stash props
```
